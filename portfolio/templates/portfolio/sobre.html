Sobre esta Aplicação

Arquitetura MVT do Django

Esta aplicação foi desenvolvida com Django e segue a arquitetura MVT: Model, View e Template.

Componentes

O Model representa os dados da aplicação e define as tabelas da base de dados. Nesta aplicação, exemplos de models são Projeto, Tecnologia, UnidadeCurricular, Competencia e Formacao.

A View recebe o pedido do utilizador, consulta a base de dados através dos models e envia os dados para o template.

O Template é responsável por apresentar os dados em HTML, criando as páginas visíveis no navegador.

Fluxo de informação

O utilizador acede a uma URL. O Django procura essa rota no ficheiro urls.py e chama a view correspondente. A view recolhe os dados nos models e envia-os para o template, que gera a página HTML apresentada no browser.

Desenho da arquitetura MVT

Modelação

A modelação da aplicação foi construída para representar o meu percurso académico, os projetos desenvolvidos, as tecnologias utilizadas, as competências adquiridas, formações, TFCs e o processo de desenvolvimento.

Diagrama Entidade-Relação em papel

Exemplo de relação no models.py

class Projeto(models.Model):
    unidade_curricular = models.ForeignKey(UnidadeCurricular,on_delete=models.CASCADE,related_name='projetos',null=True,blank=True)
    titulo = models.CharField(max_length=100,blank=True,default='')
    descricao = models.TextField(blank=True)
    tecnologias = models.ManyToManyField(Tecnologia,related_name='projetos',blank=True)

class Tecnologia(models.Model):
    nome = models.CharField(max_length=100)
    tipo = models.CharField(max_length=20,choices=TIPO_CHOICES,default='outra')

Esta relação entre Projeto e Tecnologia é do tipo muitos-para-muitos, porque um projeto pode usar várias tecnologias e uma tecnologia pode estar presente em vários projetos.
